[ 프로젝트 개요 ]
--------------------------------------------------------------------------------
故 정주영 현대그룹 회장 서거 25주기 추모 미디어월 방명록 시스템.
방문자가 모바일로 메시지를 작성하면 대형 스크린 월에 실시간으로 표시됨.

3개의 화면:
  - /wall   : 대형 스크린용 메시지 버블 표시 (WallScene)
  - /submit : 방문자용 모바일 메시지 작성 폼 (SubmitPage)
  - /admin  : 관리자용 메시지 관리 (AdminForm) — 비밀번호: HD12345

기술 스택:
  - 프론트: React 19 + Vite 5 + Socket.io-client
  - 백엔드: Express + Socket.io + better-sqlite3 (Node.js ESM)
  - DB: SQLite (messages.db, 프로젝트 루트에 위치)
  - 라우팅: React Router v6
  - 스타일: 인라인 스타일 (Tailwind는 설치되어 있으나 거의 미사용)

주요 파일 구조:
  server/
    index.js          — Express + Socket.io 서버 진입점 (포트 3001)
    db.js             — SQLite 초기화 및 쿼리
    socket.js         — Socket.io 이벤트 (new_message, message_deleted)
    routes/
      ingest.js       — POST /api/messages/ingest (X-Ingest-Token 인증)
      messages.js     — GET/DELETE /api/messages
      kakaowork.js    — 카카오워크 웹훅 수신
  src/
    App.jsx           — React Router 라우팅 설정
    services/socket.js         — Socket.io 클라이언트 싱글톤
    components/
      admin/AdminForm.jsx      — 관리자 페이지
      submit/SubmitPage.jsx    — 방문자 제출 폼
      wall/WallScene.jsx       — 대형 스크린 월
      wall/BubbleCanvas.jsx    — 메시지 버블 애니메이션
  vite.config.js      — /api, /socket.io → localhost:3001 프록시

환경변수 (.env 파일 직접 생성 필요, .env.example 참고):
  PORT=3001
  INGEST_TOKEN=dev-token         (개발용; 실제 배포 시 변경)
  KAKAOWORK_SECRET=...
  VITE_API_URL=                  (개발 시 빈값 → vite proxy 사용)

개발 서버 실행:
  npm install
  npm run dev
  → 프론트: http://localhost:5173
  → 백엔드: http://localhost:3001 (concurrently로 동시 실행)

[다음 1] 백엔드 서버 배포 (필수 — Firebase는 프론트만 호스팅 가능)
  Express + SQLite + Socket.io 서버를 별도 서버에 배포해야 함.
  추천 플랫폼 (무료 티어 있음):
    A) Railway   (https://railway.app)  — 가장 간단, Git 연동
    B) Render    (https://render.com)   — 무료 플랜 있음, sleep 있음
    C) Google Cloud Run                — Docker 이미지 필요, 안정적

  배포 전 체크리스트:
    - server/index.js의 CORS origin을 Firebase 도메인으로 제한
      현재: cors({ origin: '*' })
      변경: cors({ origin: ['https://YOUR_PROJECT.web.app', 'https://YOUR_PROJECT.firebaseapp.com'] })
    - SQLite는 영구 스토리지(볼륨 마운트) 필요 — Railway/Render에서 Volume 설정
    - 환경변수 서버에 설정: PORT, INGEST_TOKEN, KAKAOWORK_SECRET
    - 배포 후 VITE_API_URL에 해당 URL 입력

[다음 2] Firebase 프론트엔드 실제 배포
  실행 순서:
    1) npm install -g firebase-tools
    2) firebase login
    3) .firebaserc에서 YOUR_FIREBASE_PROJECT_ID를 실제 프로젝트 ID로 교체
    4) .env.production 파일 생성: VITE_API_URL=https://your-backend.com
    5) npm run deploy
       (= vite build && firebase deploy --only hosting)
  확인: /wall, /submit, /admin 직접 URL 접근 시 모두 정상 동작해야 함

[ 검증 방법 ]
--------------------------------------------------------------------------------

로컬 개발 환경에서:
  1) npm run dev 후 localhost:5173/submit → 모바일(375px) 개발자도구 확인
  2) localhost:5173/admin → 비밀번호 화면 표시, HD12345 입력 후 진입
  3) localhost:5173/admin → 375px 뷰에서 탭 전환(메시지 입력/목록) 확인
  4) npm run build → dist/ 폴더 생성 확인 (에러 없어야 함)

Firebase 배포 후:
  5) https://YOUR_PROJECT.web.app/wall   → 메시지 버블 표시
  6) https://YOUR_PROJECT.web.app/submit → 메시지 작성 및 전송
  7) https://YOUR_PROJECT.web.app/admin  → 비밀번호 게이트 → 관리 화면
  8) 직접 URL 입력(딥링크) → 각 라우트 정상 동작 (firebase.json rewrites 덕분)

[ 주의사항 ]
--------------------------------------------------------------------------------

- node_modules/ 는 압축에 포함되어 있지 않을 수 있음 → npm install 먼저 실행
- messages.db 는 SQLite 파일로 실제 메시지 데이터 포함 — 운반 시 백업 권장
- .env 파일은 .gitignore에 포함됨 — 새 PC에서 .env.example 참고하여 직접 생성
- 비밀번호 HD12345 는 AdminForm.jsx 내 ADMIN_PASSWORD 상수에 하드코딩됨
- Firebase 무료 Spark 플랜: 호스팅 10GB/월, 충분히 커버 가능
